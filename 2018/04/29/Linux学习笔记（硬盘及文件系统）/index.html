Linux学习笔记（硬盘及文件系统）

发表于 2018-04-29

1. 硬盘

1.1 硬盘的技术指标

  • 主轴转速：硬盘盘片在一分钟内所能完成的最大转速。
  • 分类：绿盘（监控） 蓝盘（普通电脑，I/O比绿盘好） 黑盘（游戏盘，家用 7200r/m） 红盘（企业）。
  • 平均寻道时间：指磁头从得到指令到寻找到数据所在磁道的时间，它描述硬盘读取数据的能力。
  • 数据传输率：指的是从用盘缓存向外输出数据的速度（MB/s）。
  • 高速缓存：缓存是数据的临时寄存器，主要用来缓解速度差和实现数据预存取等。
  • 单碟容量：指每张碟片的最大容量，这是反应硬盘综合性能的指标。

1.2 硬盘接口：

  • SCSI：商业级的关键数据的大容量存储。
  • SAS：可以支持SAS和SATA磁盘。
  • SATA：非关键数据的大容量存储。
  • PATA：（IDE接口，很多针）。

2. 静态存储

2.1 主引导记录

  • MBR位于硬盘的0磁道0柱面1扇区（512字节）
    • 硬盘操作系统的硬盘引导程序（446字节）
    • 硬盘分区表（64B）
    • 最后两个字节“55，AA”是分区的结束标志
  • MBR是由分区程序（fdisk）产生的
    • 不依赖于任何OS
    • 硬盘引导程序是可以改变的，从而实现多系统共存

2.2 磁盘分区

  • 指向附加分区描述符的扩展分区
  • 内核最多支持分区数：
    • IDE驱动器63
    • SCSI 15

2.3 分区类型

  • 分类：主分区 扩展分区
    • 主分区不可再分
    • 扩展分区可以再分

2.4 两种硬盘的存储方式

  • 基本硬盘存储
    • 在基本磁盘上存储数据需要在磁盘上创建主分区、扩展分区和逻辑分区，然后对这些分区进行管理。
  • 动态硬盘存储
    • 在动态磁盘上存储数据需要在磁盘上创建动态卷，然后对这些卷进行管理。
  • 物理分区
    • PV（物理卷）
    • 多个PV —> VG（卷组）
    • 随机抽取小于等于卷组的容量 —> LV（逻辑卷）

2.5 分区管理工具

ls /dev/sda -l     #查看磁盘分区
fdisk -l           #查看磁盘分区的详细信息
  • 常用分区工具
    • fdisk
    • sfdisk
    • GNU parted－高级分区操作（创建、复制、调整大小等等）
  • partproba #重新设置内存中的内核分区表版本
    • 在虚拟机中添加磁盘后，磁盘的命名方式为sda-z

2.6 磁盘分区实验步骤

1.为Linux添加一块硬盘

2.使用fdisk为新添加的磁盘进行分区

fdisk -l              #找到新添加的磁盘
fdisk /dev/sdb1       #进入fdisk分区工具的交互模式
p                     #查看分区信息
d                     #删除一个分区
n                     #新建一个分区
    p                 #主分区,默认为主分区
    e                 #扩展分区
    分区号             #默认
    分区起始位置        #没有分区时，默认为2048，已有分区时，默认为上一个分区最后的位置+1
    +2G               #加2G （blocks/1024/1024）
    p                 #保存之前再次查看
    w                 #保存
t                     #改变分区类型
l                     #查看分区类型
83                    #'Linux'格式，system类
w                     #保存分区后退出
q                     #不保存退出

3.格式化分区

mkfs.xfs /dev/sdb1  #敲mkfs.后按两下tab，可以看到更多可选分区类型

4.挂载分区，使用分区

mkdir /mnt/learn  #为分区创建挂载点
mount /dev/sdb1 /mnt/learn  #将sdb1分区挂载到learn文件夹下 ，但要先格式化分区，关机失效

5.保存分区表

partprobe

2.7 静态分区的缺点

  • 在安装Linux的过程中如何正确地评估各分区大小是一个难题，因为系统管理员不仅要考虑当前某个分区的容量，还要预见该分区以后可能需要的容量的最大 值。
  • 某个分区空间耗尽时，通常的解决方式为：
    • 使用符号链接—-破坏了Linux文件系统的标准结构
    • 使用调整分区大小的工具—-必须停机一段时间进行调整
    • 备份整个系统、清除硬盘、重新对硬盘进行分区，然后恢复到新分区

2.8 卸载分区

umount /dev/sdb1  #删除挂载  [设备名|挂载点]

自动挂载

vim /etc/fstab
    /dev/sdb1   /mnt/learn  xfs defaults    0   0  :wq     #使sdb1分区自动挂载到learn中，开机启动

使用blkid获取块设备uuid号进入etc/fstab进行挂载

blkid /dev/sdb1

2.9 测试自动挂载

mount -a            #查看挂载是否成功
mount               #显示挂载详情

3. 动态存储

3.1 LVM相关概念

  • LVM（逻辑盘卷管理，Logical Volume Manager）
    • Linux环境下对盘卷进行方便操作的抽象层。
    • 建立在硬盘和分区之上的一个逻辑层，来为文件系统屏蔽下层磁盘分区布局，从而提高磁盘分区管理的灵活性。
    • 允许在多个物理设备间重新组织文件系统，包括重新设定文件系统的大小。

3.2 使用LVM优点

  • 轻松管理磁盘分区。
  • 可以在卷组中随意创建逻辑卷，并进一步在逻辑卷上创建文件系统。
  • 可以方便的调整存储卷组的大小，并且可以对磁盘存储按照组的方式进行命名、管理和分配。

3.3 LVM结构图

  • 物理分区变成物理卷；
  • 物理卷任意组合成卷组；
  • 卷组划分成若干个逻辑分区；
  • 逻辑分区进行格式化，并挂载到文件中。

3.4 LVM术语–物理卷

  • 在LVM结构中处于最底层。
  • 可以是整个硬盘、硬盘上的分区或者从逻辑上与磁盘分区具有相同功能的设备（如RAID）。
  • LVM的基本存储逻辑块，包含与LVM相关的管理参数。

3.5 物理区域

  • 每一个物理卷被划分为基本单元（PE）。

3.6 卷组

  • 建立在物理卷之上，由一个或多个物理卷组成；
  • 创建后，可以动态添加物理卷组，在卷组上可以创建一个或多个“LVM分区”（逻辑卷）；
  • 一个LVM系统中可以有一个或多个卷组；
  • 类似于非LVM系统中的物理硬盘。

3.7 逻辑卷

  • 建立在卷组之上，是从卷组中“切出”的一块空间；
  • 创建之后，其大小可以伸缩；
  • 类似于非LVM系统中的硬盘分区，在逻辑卷之上可以建立文件系统（如/home或者/usr）。

3.8 逻辑区域

centOS-root   #根分区

3.9 交互界面

  • lvm
    • pvs #显示物理卷信息
    • vgs #显示卷组的信息
    • lvs #显示逻辑卷信息
    • pvdisplay #显示物理卷属性
    • vgdisplay #显示卷组的属性
    • lvdisplay #显示逻辑卷属性

3.10 创建物理卷、卷组、逻辑卷

存在两个分区/dev/sdb1 /dev/sdc1

pvcreate /dev/sdb3                     #将分区sdb3分区改为物理卷
vgcreat eagle /dev/sdb3 /dev/sdc1      #用sdb3和sdc1两个物理卷创建名为eagle的卷组
lvcreate -L 3G -n systemclass eagle    #使用eagle的卷组创建一个大小为3G的名为systemclass的逻辑卷

注：逻辑卷应该是包含在卷组中。

3.11 查看卷信息

lvs                            #查看创建关系
pvdisplay [<物理卷设备名>]       #查看物理卷信息
vgdisplay [<卷组名>]            #查看卷组信息
lvdisplay [<逻辑卷卷设备名>]     #查看逻辑卷信息

3.12 挂载逻辑卷

临时挂载：

mke2fs -t systemclass /dev/eagle/systemclass          #格式化逻辑卷
mount /dev/eagle/systemclass /mnt                     #把卷组的一个逻辑卷systemclass挂载到/mnt目录上。

开机自动挂载：

vim /etc/fstab
末行添加一条挂载信息：/dev/eagle/systemclass(绝对路径或者UUID)    /mnt    ext4    default    0 0

查看逻辑卷UUID：

blkid                    #查看UUID

注：逻辑卷要先格式化 才能产生UUID。

3.13 扩展逻辑卷

1.若卷组中无剩余空间，首先扩展卷组

1).添加硬盘，在磁盘上创建8e分区
2).在分区上创建物理卷
3).将物理卷添加到卷组中

vgextend <卷组名> <物理卷设备名> [...]
vgextend eagle /dev/sdc2                   #动态改变大小
vgs                                        #查看是否还有空余

2.若卷组中有剩余空间，扩展卷组中的逻辑卷

lvextend <-L +逻辑卷增量> <逻辑卷设备名>
lvextend <-l +PE值> <逻辑卷设备名称>
lvextend -L 1G /dev/eagle/system_class     #扩展逻辑卷

3.对已扩展的逻辑卷中的文件系统进行容量扩展

resize2fs <分区或逻辑卷设备名>

4.将两个物理卷扩展到已存在的eagle卷组中 

vgextend eagle /dev/sdb1 /dev/sdb2

5.在eagle卷组中扩展system_class逻辑卷，扩展大小为6GB 

lvextend -L +6G /dev/eagle/system_class

6.为home文件系统（ext3）扩充容量

resize2fs -f /dev/eagle/system_class

7.使用umount命令卸载文件系统

8.使用e2fsck命令检查文件系统

9.使用resize2fs命令缩减文件系统容量

10.缩减逻辑卷

lvreduce <-L -逻辑卷增量> <逻辑卷设备名称>
lvreduce <-l -PE值> <逻辑卷设备名称>

4. 文件系统

4.1 定义

  • 文件系统是包括在一个磁盘（硬盘、光盘及其他存储设备）上的目录结构，一个磁盘设备可以包含一个或多个文件系统。

4.2 Linux的文件系统结构

  • Linux下的所有文件和目录以一个树状的结构组织构成了 Linux 中的文件系统。
    • Linux文件系统标准（Linux File System Standard，FSSTND）
    • 文件系统层次结构标准（File System Hierarchy Standard，FHS）

4.3 Linux的多种文件系统

  • Linux的内核采用了虚拟文件系统（Virtual File System，VFS）技术，因此Linux可以支持多种不同的文件系统类型。
  • Linux可以支持的文件系统：
    • UNIX类的文件系统：HFS、XFS、JFS、Minix FS、UFS等。
    • NFS网络文件系统。
    • NTFS和vfat(FAT32)文件系统。

4.4 Linux的日志文件系统

  • Linux支持的日志文件系统：
    • ext3/ext4
    • JFS（IBM）
    • XFS（SGI）
    • Reiserfs

4.5 挂装文件系统–mount命令

mount  [选项]  [<分区设备名>]  [<挂载点>]      #例如：mount -t ext3 /dev/sdb1 /opt
  • 常用选项
    • -t：<文件系统类型>：指定文件系统类型
    • -r：使用只读方式来挂载
    • -a：挂装/etc/fstab文件中记录的设备
    • -o：iocharset=cp936：使挂装的设备可以显示中文文件名
    • -o：loop：使用回送设备挂装ISO文件和映像文件

4.6 /etc/fstab文件的格式

  • 每一行说明一个文件的挂载信息
  • 第一个字段：分区或Lv
  • 第二个字段：挂载点
  • 第三个字段：文件系统类型
  • 第四个字段：挂载选项
    • defaults：使用rw，suid …… 挂装设备
    • acl/noacl：
    • async：非同步方式执行文件系统的输入输出
  • 第五个字段：备份频率
  • 第六个字段：检查顺序

5. 交换空间

1.交换分区和交换文件

1).Linux系统使用交换分区/文件实现虚拟内存技术，它是系统RAM的补充。

2).进程长时间占用内存空间不活动时，将这部分内存交换到交换分区中，当进程活跃时，再将其读入到真实的物理内存中。

首先创建交换分区或者文件 
mkswap /dev/                        #创建swap文件
vim /etc/fstab                      #添加条目进行挂载
UUID swap swap defaults 0 0 
swapon -a                           #激活

3).生成交换文件 

dd if=/dev/zero of=/swapfile bs=1024 count=X

6. Ext 2/3/4文件系统管理

1.创建文件系统

1).前端命令mkfs的格式

mkfs -t <fstype> -c  <分区设备名>           #例：mkfs -t ext3 -c /dev/hda2
    -t fstype：指定文件系统类型
    -c：建立文件系统前先检测有无坏块

2.检查文件系统

1).fsck是操作系统扫描文件系统内容检查内部一致性的工具。

fsck [选项][-t 文件系统类型] <设备名> [特定文件系统的附加选项]
fsck -t ext4 /dev/sdb2            #检查文件系统，对所有问题回答“yes”
e2fsck  -p  /dev/sda5             #自动检查并修复文件系统
fsck.ext3  -f  -y  /dev/hda2      #强制进行文件系统检查，标识损坏区块，对所有问题回答“yes”

3.显示和调整文件系统属性

1).显示文件系统属性参数

tune2fs -l  <device> 
dumpe2fs -h <device>

2).可调整的文件系统属性参数：保留块 、默认挂载选项 、fsck频率。

7. 磁盘限额

1.硬盘限额配置步骤

1).启用文件系统的quota功能

vim /etc/fstab                     #添加文件系统挂载选项usrquota,grpquota。
LABEL=/home      /home         ext3        defaults,grpquota,usrquota       1  2
mount -o remount /home             #重新挂装文件系统

2).创建quota数据库并开启quota

quotacheck -cmvug/home             #创建quota数据库
quotaon -avug                      #开启quota功能

3).设置用户和组的quota

edquota                            #交互式编辑配额
setquota                           #命令式设置配额
edquota -p <protoname>             #将参考用户/组的配额复制给其他用户/组
setquota -p <protoname>            #将参考用户/组的配额复制给其他用户/组

edquota命令：

edquota [-u] [-f filesystem]<username>         #编辑指定用户的配额
edquota -g [-f filesystem] <groupname>         #编辑指定组的配额
esquota -t [-u] [-f filesystem]                #编辑指定用户的配额时限
edquota -t -g [-f filename]                    #编辑指定组的配额时限
-f filename 表示对指定的文件系统设置配额，省略时表示对所有启用quota的文件系统进行设置

setquota命令：

setquota [-u]<username><块软限制 块硬限制 inode软限制 inode硬限制><-a|文件系统>      #为指定用户设置配额
setquota -g <groupname><块软限制 块硬限制 inode软限制 inode硬限制><-a|文件系统>      #为指定组的设置配额
setquota -t [-u] <块时限inode时限><-a|文件系统>                                   #为指定用户设置配额时限
setquota -t -g <块时限inode时限><-a|文件系统>                                     #为指定组设置配额时限

注：
软限制：当到达设置值时会出现警告，但不会禁用。
硬限制：当到达设置值时会强制停止。
块：内存容量。
inode：文件个数。
若对所有启用了quota的文件系统设置配额，则使用-a；否则，若对指定的文件系统设置配额，则需要指定文件系统。

使用参考用户或组复制配额：

将参考用户protouname的限额设置复制给带设置用户username:

edquota [-u] -p <protouname><username><-a|文件系统>
setquota [-u] -p <protouname><username><-a|文件系统>

将参考组protogname的限额设置复制给待设置组groupname:

edquota -g -p <protouname><username><-a|文件系统>
setquota -g -p <protouname><username><-a|文件系统>

查看磁盘限额:

查看指定用户或者组的quota设置:

quota [-vl] [-u <username>]
quota [-vl] [-g <groupname>]
quota -q

显示指定文件系统的磁盘限额汇总情况:

repquota [-ugv] filesystem

显示所有文件系统的磁盘限额汇总信息:

repquota [-augv]
